3. Forbidden isn’t (Web Security).

“take care of your robot.”

when we connect to our given url, we only get a “Not correct, try harder.”, meaning there’s something missing that’s not letting us see whatever we’re supposed to see, so let’s dig around.

/robots.txt exists and shows this,

Contents of robots.txt

after opening that file, we get a piece of PHP code (beautified for convenience)

<?php $_0x42f35 = [@unserialize](http://twitter.com/unserialize)(@$_GET[‘_0x1337’]);
if (@$_0x42f35[‘source’] == 1)
{
 highlight_file(__FILE__);
 die();
}

let’s carefully read through this, we have the variable *$_0x42f35 *that stores an unserialized version of a value that’s gonna be passed through the parameter _0x1337.

if we read more we notice *‘source’ *which indicates that we’re gonna be passing a serialized PHP array to the param *_0x1337 *that has a key ‘source’ and a value of 1 so we can show the contents of the file included aka FILE.

this little piece of code can let us serialize our arrays,

<?php
$value = array("source" => 1);
echo serialize($value);
?>

our serialized value is a:1:{s:6:”source”;i:1;}.

meaning our current payload is: ?_0x1337=a:1:{s:6:”source”;i:1;}

this gives us the full PHP code like so

<?php 
include("flag.php");
$_0x42f35=@unserialize(@$_GET['_0x1337']);
if(@$_0x42f35['source']==1)
{
highlight_file(__FILE__);die();
}

$ـ0x4738e=@$_0x42f35['password'];
$_27f6a3="Its_easy";
$_0x41243=preg_replace("/$_27f6a3/",'',$ـ0x4738e);
if($_0x41243===$_27f6a3)
{echo $flag;}
else
{echo "Not correct, try harder.";}
?>

if we read through the unlocked second part we can see that we have 3 new variables. you can rename them to not get confused but we’ll just roll with it.

ـ0x4738e, this takes the value of the key "password" that's gonna be in our serialized array from before.

_27f6a3, this one just has a string "Its_easy".

_0x41243, and this one has a preg_replace() that's checking for the string "Its_easy" in our first variable (aka password value) and replacing it with an empty string ("").
meaning that if we put "Its_easy" as our value it's just gonna be empty after this is executed.

to escape this we can just add an “Its_easy” in the middle of another “Its_easy” so we can get our final value by removing the one in the middle lol.

our last payload is gonna be,

a:2:{s:6:"source";i:0;s:8:"password";s:16:"ItsIts_easy_easy";}
  • you can notice that I changed the source value to 0 so we don’t highlight our file since we only need to echo out the flag

after doing this we get a “WoW” and the flag is an Html comment.

Answer: FLAG{Bypa$$_!$_Aw3S0m3}.

8. Mutation (Digital Forensics).

“someone hacked the department portal and changes the visual appearance of the website, can you see the changes”.

after downloading our zip file, we get two folders *original_site *and pwned_site.

running a recursive diff against these two folders gives us the following output.

$ diff -qr original_site/ pwned_site/
> Files original_site/index.html and pwned_site/index.html differ.

meaning that every other file is the same, just like the description said, we had visual changes, if we take a look at our files, we can see that *pwned_site/index.html *has only some letters changed here and there, meaning that our flag is right in there.

doing this manually is possible but is a bit of a headache, so I wrote this little script that does that for us

f1 = open("mutation_site/original_site/index.html", "r")
f2 = open("mutation_site/pwned_site/index.html", "r")

original = f1.read().strip().replace("\n", "")
pwned = f2.read().strip().replace("\n", "")

pwned_word = ""
original_word = ""
i = 1

try:
 while True:
  if original[i] != pwned[i]:
   pwned_word += pwned[i]
   original_word += original[i]
  i += 1
except:
 print(pwned_word)
 print(original_word)

running this gives us the following output

9. KeyKey (Malware Reverse Engineering).

“Get the Key asap!!”

After downloading our zip file and compiling our code like instructed, we take a look at our compiled file in r2 (r2 is the goat istg).

just like the source code said, we get a 0x3039 and add a 1 to it which gives us a 0x303A, that value is getting compared and is outputting “The flag is XXXXX” if we’re correct, we simply convert that Hexadecimal value to decimal and get our key

303A(16) => 12346 (10)

Our flag is then the MD5 hash of our key.

Answer: a3590023df66ac92ae35e3316026d17d

*you can solve this without r2 of course but who doesn’t like r2 amirite.

10. Mr Do0om (Web Security).

“Hi I am Mr Doom, message me with two secret letters.”

Entering the website gives us a text field, we look at our source code and find this script.

for both messages we’re gonna be sending our strings split by “#” like our code says “actual_args = data.split(/#/)”

our first slice can take letters and single quotes, meaning that we can use the single quote to escape the whole thing and by adding a “;” at the end we can insert whatever we want after.

Our payload is gonna be: ‘#’;alert(‘flag1’)//

This will give us the first part of our flag, we’ll keep that on the side for now.

Second message is getting our first slice and adding it to a create function, after playing around with the create functions we can find that Comment, turns the whole thing into a comment.

Our payload is gonna be: Comment#>alert(1)

Answer: FLAG{th!5_!5_my_l0ng_Fl4g_F0r_5w33t_D0M!!}

A similar challenge was featured in the escape.alf.nu XSS challenges, you can find more insight there.

11. r34lity (Web Security).

“I love mango and i love db too.”

First thing we notice is, this challenge is screaming mongoDB, let’s explore.

Included website only gives us a “Hi”

robots.txt exists and shows two interesting files.

if ( $_GET['flag_id'] == $flagid ){ echo $flag; }

Challenge is still screaming mongoDB lol, anyway after reading this code we notice that we need a parameter flag_id with the right id to get our flag.

We also get 3 IDs that are supposed to help us out.

after some Googling around, i found that mongoDB has ObjectIDs that are meant to represent various data in mongoDB, and that these ObjectIDs are not completely random.

These values basically follow this formation.

first 4 bytes => Timestamp value, creation date of said ObjectID.
following 5 bytes => random value.
last 3 bytes => incrementing value that can ultimately be predicted.

After using a **tool** i found online, i tested our given IDs to see if they match the timestamps provided in our *l0g.hacker* file, we can notice that the logs are 7 hours late; meaning that to obtain the first part of our flag id we need to use our given timestamp, minus 7 hours.

which gives us the first part of our ID:
5cef6043----------------

Second part is a totally random value, however, we can notice that in all the provided IDs, there’s a slice thats being repeated in all 3 IDs, let’s just yoink that out of there.

Second part of our ID:
5cef60435c1fa2a0df------

Third part is the incremented part, we can notice a pattern where our IDs are going from 7 to 8 to 9 here you can safely guess it’s a, i ended up going to intruded and bruteforced from 0 to f, but you get the idea lol.

Our completed flag_id:
5cef60435c1fa2a0df9f3daa

We send a GET request with ?flag_id=5cef60435c1fa2a0df9f3daa and we get our flag.

Answer: FLAG{0bj3ct_Id$_!s_w0Nd3rFul}
